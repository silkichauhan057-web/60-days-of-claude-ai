# Day 2 — Prompt Engineering Fundamentals

**Challenge:** 60-Day Claude AI Mastery Challenge
**Phase:** 1 — Foundation & Prompt Engineering
**Focus:** Understanding the difference between a normal prompt and an engineered prompt.

## What I Learned

Prompt engineering is the practice of structuring an AI input so it reliably produces the intended output, instead of a vague or generic one. It works by removing ambiguity — specifying role, audience, format, tone, and goal — so the model doesn't have to guess what's wanted.

## Before vs After

### Before (weak prompt)
```
Write about AI.
```

**Problem:** No audience, no format, no length, no tone, no goal. The output would be generic and unfocused.

### After (engineered prompt)
```
Act as a technology writer. Write a 150-word LinkedIn post explaining how AI
is changing the job market for students, aimed at college students and
early-career professionals. Use a confident, motivating tone, include one
real-world example, and end with a question to spark engagement.
```

**Improvements made:**

| Element | Added |
|---|---|
| Role | "Act as a technology writer" |
| Audience | College students / early-career professionals |
| Format | 150-word LinkedIn post |
| Tone | Confident, motivating |
| Goal | Ends with an engagement question |

## Key Takeaway

A small change in prompt structure leads to a large change in output quality and usability. Engineered prompts turn a vague request into a precise instruction.
